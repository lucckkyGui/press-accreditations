# PressOps by OSURMO — Pakiet pilotażowy (EventOps Pilot)

> Wdrożenie PressOps na jednym realnym wydarzeniu, z raportem wartości medialnej
> dla sponsora. 8-tygodniowy program.

## 1. Zakres

Pilotaż obejmuje pełen workflow media operations dla **jednego wydarzenia**:

```
event → landing page → media submission → verification → approval →
QR pass → check-in → coverage collection → media report
```

W zakresie:
- konfiguracja wydarzenia i publicznego landingu akredytacyjnego,
- przyjmowanie i weryfikacja zgłoszeń mediów (scoring + flagi ryzyka),
- decyzje akredytacyjne (approved / approved_limited / rejected / waitlisted),
- generowanie QR pass + e-mail z decyzją,
- check-in QR na miejscu (online + tryb offline),
- zbieranie coverage po evencie (secure link dla dziennikarzy + remindery),
- **Media Coverage Report** (PDF + CSV) z wartością medialną dla sponsora.

Poza zakresem pilota: white-label, API/webhooki, integracje na zamówienie,
wallet pass, face recognition (dostępne w wyższych pakietach / post-pilot).

## 4. Wymagania od klienta

- Konto organizatora (e-mail służbowy) z rolą `organizer`.
- Dane wydarzenia: nazwa, termin, lokalizacja, pojemność, strefy dostępu.
- Branding: logo, kolory (opcjonalnie regulamin akredytacji).
- Lista typów akredytacji / poziomów dostępu specyficznych dla eventu.
- Osoba kontaktowa po stronie biura prasowego (decyzje akredytacyjne).
- (Opcjonalnie) dane sponsora do raportu wartości medialnej.
- Skonfigurowane: domena nadawcza e-mail (Resend) — lub korzystamy z naszej.

## 5. Deliverables

W trakcie pilota klient otrzymuje:
- skonfigurowane wydarzenie + publiczny landing akredytacyjny (URL do udostępnienia),
- panel weryfikacji i decyzji z media verification scoring,
- system QR pass + check-in (działa też offline),
- Coverage Board + secure linki dla dziennikarzy,
- konta i role dla zespołu (role-based access),
- krótkie szkolenie / przewodnik (demo script).

## 6. Co klient dostaje po evencie

- **Media Coverage Report (PDF)** — dokument dla sponsora: funnel akredytacji,
  frekwencja mediów, coverage performance, wzmianki sponsora, top publikacje,
  missing coverage, rekomendacje.
- **CSV** ze wszystkimi publikacjami (do dalszej obróbki).
- **Media CRM** — baza kontaktów i mediów z historią, ratingiem i wskaźnikami
  (no-show rate, coverage rate) do kolejnych wydarzeń.
- **Case study** (na podstawie szablonu) — gotowe do użycia w sprzedaży/PR.
- Rekomendacja kontynuacji (Press Pro) z wnioskami z pilota.

## 7. Sukces pilota — kryteria

- Min. 1 realne wydarzenie obsłużone end-to-end w systemie.
- Zgłoszenia mediów przyjęte przez landing (realne dane, nie demo).
- Check-in przeprowadzony na miejscu.
- Coverage zebrane i raport dostarczony sponsorowi bez ręcznej obróbki.
